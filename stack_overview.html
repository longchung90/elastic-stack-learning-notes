ES|QL — Query Language
Pipeline-based query language. Chain commands with | to filter, compute, aggregate, and shape results.
FROM · WHERE · STATS
Core pipeline. Select a source, filter rows, aggregate into groups. The pattern used in 90% of queries.
core
EVAL · CASE · TO_INTEGER
Compute new columns at query time. Workaround for mapping mistakes like response stored as text.
compute
DATE_TRUNC · DATE_FORMAT
Time bucketing and extraction. Enables hourly/daily trend charts and day-of-week analysis.
time
KEEP · DROP · RENAME
Output shaping. Control which columns appear and what they are called. Essential for readable results.
output

Elasticsearch — Storage & Search Engine
The core. Distributed document store built on Apache Lucene. Where shards, mappings, and ILM live.
Indices & Shards
Each index is split into N primary shards (Lucene instances). Shard pruning skips shards outside the time window before reading a single document.
storage
Index Mapping
The schema. Defines field types — keyword, text, date, ip, geo_point, long. Gets these wrong and you cannot fix them without a full reindex.
schema
ILM — Index Lifecycle Management
Automates hot → warm → cold → frozen → delete transitions. Runs every 10 minutes. No human action needed once configured.
lifecycle
Aliases & Templates
Alias = the stable write address that never changes across rollovers. Template = the blueprint applied to every new matching index automatically.
routing
↕
🔀
Logstash — Ingest Pipeline
Sits between your data sources and Elasticsearch. Parses, enriches, filters, and routes events before indexing.
Input plugins
Receives data from Filebeat (port 5044), Kafka, syslog, HTTP, S3, and more. Each source is a named input block.
ingestion
Filter plugins — grok · mutate · date
grok parses raw log lines into structured fields. mutate renames and converts types. date sets @timestamp from the event time, not ingest time.
transform
Output — Elasticsearch plugin
Ships events to ES via the alias name. manage_template: false is critical — prevents Logstash from overwriting your explicit mapping.
output
Dead-letter queue
Documents that fail parsing or indexing land here for inspection. Prevents silent data loss on malformed events.
reliability
↕
📄
Filebeat — Lightweight Agent
Runs on each server. Tails log files and ships raw lines to Logstash or directly to Elasticsearch.
File input / harvester
Monitors log file paths. Tracks read position so it never re-ships the same line after a restart. Backpressure-aware — slows down if Logstash is busy.
collection
Modules
Pre-built configs for common log formats — Nginx, Apache, MySQL, system logs. Automatically sets field names to ECS standard.
configuration
